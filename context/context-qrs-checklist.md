# context-qrs-checklist.md (부적합 대응 및 품질 협업 도메인 안전 체크리스트)

이 체크리스트는 QRS 도메인 관련 파일(`service/qrs_df.py`, `queries/qrs_query.py` 등)의 개발이나 워크플로우 로직 패치 시, 머지 전 완벽히 합치되어야 하는 **안전성 오퍼레이션 진단표**입니다.

---

## 1. 워크플로우 상태 관리 무결성 (Workflow & Security Guard)
- [ ] 신규 상태 추가나 변경 시, 상태 천이 전후의 정당성을 보증하는 비즈니스 벨리데이션 체크 로직이 온전히 포함되어 있는가?
- [ ] 권한 그룹(Admin, Contributor, Viewer)에 맞는 화면 노출 제어 및 수정 비활성화(`disabled=True` 속성) 처리가 Streamlit 위젯에 확실히 탑재되었는가?
- [ ] 타인의 미결 워크시트를 다른 사용자가 권한 없이 임의 수정하지 못하도록 원 소유자 매핑(Owner Verification) 검증이 진행되는가?

---

## 2. SQLite 오디팅 동기화 및 롤백 (Audit & Sync Logging)
- [ ] 상태의 변경(Status Update) 이벤트가 발생할 때마다, SQLite 로그 핸들러(`core/db/sqlite_utils.py` 등)를 경유해 변경자 ID, 일시, 이전 상태, 신규 상태가 영속 기록되도록 설계되었는가?
- [ ] 데이터베이스 다중 쿼리 수정(Multi-Query Update) 도중 중도 실패 시 전체 수정 상태가 이전 상태로 자동 롤백되도록 트랜잭션 원자성이 설계되었는가?

---

## 3. 진행 현황 대시보드 시인성 (UX Visibility Guard)
- [ ] 작성 후 미결 상태로 남아 있는 장기 미처리 부적합 건에 대해 대시보드 화면상에서 경고성 색상(예: 세련된 파스텔톤 오렌지/레드 계열 배경)으로 강조하고 있는가?
- [ ] 부적합 진행 현황을 간편하게 집계할 수 있도록 상태별 카운트 요약 카드(`st.metric`)가 화면 상단에 정렬되어 배치되는가?
