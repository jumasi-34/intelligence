# alert-mail.summary.md (알람 및 메일 자동화 배치 도메인 요약)

이 시스템에서 'Alert & Mail'은 품질 이상 계측 건을 백그라운드로 수시 모니터링하여 공장 책임자 및 대책 부서에 실시간 자동 경보를 주입하고, 주기적인 주간 품질 보고 대시보드 요약본을 자동으로 전송하는 **자동화 및 통지 허브**입니다. 무한 메일 오발송 루프나 자격 증명 유실 방지가 핵심인 오퍼레이션 영역입니다.

---

## 1. 도메인 규칙 및 불변 조건 (Invariants)
- **메일 오발송 무한 루프의 절대 차단**:
  - 알림 유발 조건 탐색 배치 작업 도중 예외가 발생하더라도 메일 전송 블록(`send_mail()`)이 무한 재시도 루프에 빠져 고객사의 메일 서버가 스팸 차단당하거나 비용 폭탄을 맞지 않도록 최대 시도 횟수 제한(`MAX_RETRIES = 3`)을 강제해야 합니다.
- **주간/실시간 모니터링 조건의 연속성**:
  - 주간 자동 통지는 무조건 사전에 정제 완료된 SQLite 집계 테이블(`ops.db` 등)을 경유하여 검증된 배치 데이터에 한해서만 발송이 일어나야 합니다. 직접 Databricks 실시간 조회 쿼리를 대량 메일러 루프 안에서 가동하는 행위를 엄격히 불허합니다.

---

## 2. 민감 소스 파일 (Sensitive Files)
- `core/db/mailer.py` (SMTP 메일링 서버 포트 연동 및 이메일 HTML 레이아웃 조립)
- `core/operate/send_mail.py` (메일 수신자 식별, 헤더 및 첨부파일 바인딩 인터페이스)
- `automation/automail_cqms_ongoing.py` (CQMS 지속 이상 계측 배치 모니터 및 주간 스케줄 기반 자동 발송 실행부)

---

## 3. 절대 금지 행동 (Prohibited Actions)
- SMTP 서버 접속 비밀번호나 시스템 인증 패스워드를 `mailer.py` 등 파이썬 코드 내부에 평문 문자열 상수로 하드코딩 기재해두는 일.
- 수신자 목록(`RECEIVERS`) 컬럼을 소스 코드에 명시적으로 하드코딩 결합하여 인사이동이나 인입 공장 변경 시 매번 코드가 변경되도록 방치하는 행동 (SQLite 설정 테이블이나 `.env`를 통한 조회로 통일).

---

## 4. 필수 테스트 시나리오 (Required Test Scenarios)
- **수신자 부재 방어 테스트**: 통지 대상자의 연락처나 수신 주소 목록이 DB 오류로 비어 있을 때, `st.error` 또는 배치 로그 상에 경고를 즉각 남기고 메일 엔진 발송 루틴을 깨끗이 탈출(Early Exit)하여 크래시를 차단하는가?
- **메일 헤더 인코딩 검증**: 한국어, 영어, 특수문자가 혼합된 불량 자재명이나 오류 유형 문자열 제목이 이메일 수신함에서 깨지지 않고 한글 인코딩(`MIMEText`, `UTF-8`)에 의해 깨끗이 파싱 표출되는가?
- **SMTP 타임아웃 예외 제어 테스트**: 수신자측 메일 방화벽이나 SMTP 연결이 타임아웃 지연(예: 10초)으로 뻗을 때, 배치 데몬 전체가 메모리 릭(Memory Leak)으로 죽지 않고 연결 실패를 SQLite 오디팅 로그에 온전히 기재한 뒤 안전 종료되는가?
