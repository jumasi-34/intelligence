# checklist-git.md (L1 Git 형상관리 및 동기화 정책 체크리스트)

본 문서는 프로젝트 형상 관리 일관성을 유지하고 변경 추적 및 자산의 안전한 동기화를 완벽히 수행하기 위한 Git 커밋 및 동기화 정책 체크리스트입니다.

---

## 1. 커밋 메시지 표준 규칙
- [ ] **한국어 엄수**: 모든 Git 커밋 메시지(제목 및 본문)가 한국어로 구체적이고 정중하게 명시되었는가?
- [ ] **대괄호 태그 필수 포함**: 커밋 헤더 맨 앞부분에 작업 성격에 가장 정확히 부합하는 대괄호 태그(`[TAG]`)가 필히 포함되었는가?
  - `[FEAT]`: 신규 기능 개발 및 추가
  - `[FIX]`: 버그, 오류, 예외 상황 수정
  - `[REFACTOR]`: 비즈니스 로직 수정 없는 구조 개선
  - `[STYLE]`: 코드 작동에 무관한 스타일/포맷팅 변경
  - `[DOCS]`: 문서 작성 및 수정
  - `[RULE]`: 규칙 제정 및 에이전트 행동 가이드 변경
  - `[TEST]`: 테스트 하네스 및 검증 코드 추가
  - `[CHORE]`: 빌드 설정, 단순 환경 설정 등
- [ ] **원자적 커밋 단위**: 하나의 커밋이 하나의 논리적 변경 사항(원자적 작업 단위)만을 포함하고 있는가?

---

## 2. 동시 푸시 및 로컬 동기화 단방향 정책
- [ ] **동시 푸시 수칙 (Dual Push Policy)**: 메인 저장소에 대한 push 요청 시, 중첩 연결된 독립 레포인 `intelligence/` 디렉터리 내의 변경 사항을 누락 없이 순차적으로 먼저 커밋 및 동시 푸시를 수행했는가?
- [ ] **로컬 애셋 동기화 방향 (Push Only)**:품질 매뉴얼, 설계 도면, 이미지 등 무겁고 정적인 로컬 자원이 위치한 `local.assets/` 디렉터리에 대해 항상 로컬(WSL)에서 원격 서버(Ubuntu)로의 **Push 단방향 동기화**만 수행하였는가? (원격에서 로컬로의 Pull은 절대 금지)
- [ ] **로컬 데이터 동기화 방향 (Pull Only)**: 로그 및 SQLite DB 등이 포함된 운영 데이터 공간인 `local.data/` 디렉터리에 대해 항상 원격 서버(Ubuntu)에서 로컬(WSL)로의 **Pull 단방향 동기화**만 수행하였는가? (로컬에서 원격으로의 Push는 절대 금지)
- [ ] **동기화 스크립트 전용 호출**: 모든 Rsync 동기화 처리가 지정된 자동화 쉘 스크립트(`automation/sync_local_assets.sh` 또는 `sync_local_data.sh`)를 통해서만 실행되는가?
- [ ] **접속 인증 정보 은닉**: 원격 서버의 접속 및 로그인 인증 세부 정보가 소스코드에 평문으로 하드코딩되지 않고, 루트 디렉터리의 `.env` 파일에 안전하게 정의 및 은닉되어 로드되는가?
