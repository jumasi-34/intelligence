# intelligence/runs/ (에이전트 실행 이력 영속 저장소)

이 디렉터리는 CQ-BI 시스템 상에서 가동된 모든 자율 에이전트의 구동 세션 이력(Runs)과 7대 핵심 아티팩트가 실시간으로 보관되는 **영속 로깅 트랙 데이터 아카이브**입니다.

---

## 1. 아티팩트 보관 폴더 구조

각 실행은 고유한 실행 ID 포맷인 `run_YYYYMMDD_HHMMSS_[hash]` 아래에 단독 폴더로 영속 분리 저장됩니다.

```
ai/runs/
├── README.md                      # 본 가이드라인 파일
│
├── run_20260530_185937_c91a3b/     # [실행 예시 디렉터리]
│   ├── task.md                    # 에이전트 실행 대상 피처 및 버그 명세서
│   ├── context-manifest.yaml      # 주입된 도메인 룰 및 코딩 표준 파일 경로 목록
│   ├── plan.md                    # 코딩 수정 가동 전 에이전트가 도출한 실행 계획 및 결정 근거
│   ├── diff.patch                 # 수정한 소스 코드의 로컬 Git Diff 패치 파일
│   ├── test.log                   # 빌드 및 `make verify` 구문 검사 터미널 출력 로그
│   ├── review.md                  # qa-reviewer가 정적 분석을 통해 피드백한 리뷰서
│   └── risk.md                    # ops-release-guard가 측정한 배포 안정성 보고서
│
└── ...
```

---

## 2. 7대 핵심 아티팩트 세부 규격 (Artifact Specification)

### [1] `task.md`
- 에이전트가 시작 단계에서 할당받은 작업 목표를 마크다운 형식으로 기록합니다.
- 대상 파일, 오류 메시지 원본, 작업자 세션 정보 및 타겟 도메인 정보가 포함됩니다.

### [2] `context-manifest.yaml`
- 에이전트 추론 정확성을 추적하기 위해 당시 로드된 컨텍스트 파일의 체크섬(MD5 Hash)과 물리 파일 경로 목록을 기록하는 구조화된 매니페스트 파일입니다.

### [3] `plan.md`
- 에이전트가 실제 소스 코드 파일을 변경하기 전, 실행 계획과 결정 근거를 요약해 기록한 파일입니다. 내부 추론 원문이나 민감 데이터가 아니라 어떤 클래스와 메소드를 왜 건드릴 것인지 남기는 용도입니다.

### [4] `diff.patch`
- 에이전트가 완성해 낸 파일 차이점(`git diff` 포맷)을 온전히 저장합니다. 추후 코드 변경 복구(Rollback)가 요구되거나 인간 시니어의 승인이 필요할 때 다이렉트로 patch를 적용해 롤백하거나 적용할 수 있습니다.

### [5] `test.log`
- 에이전트 기동 직후에 `tests/verify_code.py` 및 도메인 지정 하네스 테스트를 트리거한 터미널 출력 결과물입니다. `PASSED` 여부와 린트 경고 라인 등이 가감 없이 영속 보관됩니다.

### [6] `review.md`
- `qa-reviewer` 에이전트가 `diff.patch`를 전수 스캔하여 결함, 보안 취약점(하드코딩 패스워드 등) 및 테스트 누락 여부를 정적 진단하고 작성한 리뷰 보고서입니다.

### [7] `risk.md`
- `ops-release-guard` 에이전트가 이 코드 변경 사항이 운영 프로덕션 시스템에 배포될 때 유발할 수 있는 잠재 리스크를 종합 진단(High / Medium / Low)하고 차단 사유를 명시한 보고서입니다.

---

## 3. 저장소 관리 준수 사항
- 해당 디렉터리 내에 생성되는 `run_` 하위 아티팩트들은 오디팅을 위한 불변 기록물이므로, **인간 개발자 및 에이전트의 임의 수정 및 파일 임의 삭제를 금지**합니다.
- 디스크 디바이스 용량 관리를 위해, 6개월이 경과된 고구버전 실행 이력 디렉터리는 `ops-release-guard` 또는 시스템 어드민의 자동 회전 정책에 의해 압축 백업 보관 처리됩니다.
